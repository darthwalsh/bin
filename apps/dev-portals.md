# Developer Documentation Portals: Backstage TechDocs vs Stoplight

Mental model: **Backstage = portal docs tied to services** · **Stoplight = API-first interactive docs hub**


| Feature                          | Backstage TechDocs                                                                                                                                                                                                                                               | Stoplight                                                                                                                                                                     |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Primary goal**                 | Docs-as-code inside an internal developer portal; ties docs to catalog entities (services, teams)                                                                                                                                                                | API contract publishing; generate interactive reference docs from OpenAPI + markdown guides around them                                                                       |
| **Source format**                | Markdown files co-located in repo; requires `mkdocs.yml`                                                                                                                                                                                                         | OpenAPI (YAML/JSON) for reference docs; [Stoplight Flavored Markdown](https://docs.stoplight.io/docs/platform/b591e6d161539-stoplight-flavored-markdown-smd) (SMD) for guides |
| **Build pipeline**               | `mkdocs` → Material theme → static HTML; CI publishes via [`techdocs-cli`](https://backstage.io/docs/features/techdocs/cli/) to cloud storage (S3, GCS, Azure Blob)                                                                                              | Webhook or CLI trigger; Stoplight ingests repo content (Markdown, YAML, JSON); controlled by `.stoplight.json` / `.stoplightignore`                                           |
| **Publishing a new file**        | Add `.md` file to repo, add to `mkdocs.yml` `nav`, push → CI regenerates + publishes to storage bucket                                                                                                                                                           | Push `.md` or OpenAPI file to repo branch; Stoplight picks it up on next sync (webhook or manual)                                                                             |
| **Catalog / discovery**          | Tight: docs surface on the service's Backstage catalog page; searchable across the portal                                                                                                                                                                        | API catalog view; hub-wide [search](https://stoplight.io/api-documentation) across endpoints, schemas, guides                                                                 |
| **Search**                       | Built-in full-text search via TechDocs search plugin; integrates with Backstage global search                                                                                                                                                                    | Hub-wide search over endpoints, reference docs, and schemas                                                                                                                   |
| **Versioning**                   | No native doc versioning UI; use git branches — each branch must be published separately                                                                                                                                                                         | [Multiple branch support](https://blog.stoplight.io/stoplight-v2-0-0-release); assign display names to branches; version dropdown in UI                                       |
| **Interactive API "try it out"** | Not native; possible via OpenAPI viewer plugins                                                                                                                                                                                                                  | Core feature — try-it-out UI generated from OpenAPI via [Elements](https://stoplight.io/elements)                                                                             |
| **Code samples**                 | Not native                                                                                                                                                                                                                                                       | Auto-generated [code samples](https://stoplight.io/api-documentation) in curl, Python, Ruby, Java, etc.                                                                       |
| **Mock server**                  | Not native                                                                                                                                                                                                                                                       | [Prism](https://stoplight.io/open-source/prism) open-source mock server, OpenAPI-powered                                                                                      |
| **Diagrams — Mermaid**           | Via addon [`backstage-plugin-techdocs-addon-mermaid`](https://www.npmjs.com/package/backstage-plugin-techdocs-addon-mermaid); requires `mkdocs-techdocs-core >= 1.0.2` + Backstage app config                                                                    | [Supported natively since 2021](https://blog.stoplight.io/a-technical-writers-experience-using-mermaid-in-stoplight) inside SMD fenced code blocks                            |
| **Diagrams — PlantUML**          | Supported via [`plantuml-markdown`](https://pypi.org/project/plantuml-markdown/) included in `mkdocs-techdocs-core`; ⚠️ `svg_object` format blocked for security — use `svg_inline` instead ([docs](https://backstage.io/docs/features/techdocs/how-to-guides/)) | ⚠️ **Unverified** — no official Stoplight doc confirms native PlantUML rendering; workaround: render to SVG/PNG externally and embed as image                                 |
| **Custom theme / branding**      | Material for MkDocs theme; limited customization within Backstage portal chrome                                                                                                                                                                                  | [Custom branding and domain](https://stoplight.io/api-documentation); white-label developer hub                                                                               |
| **Access control**               | Backstage auth + permissions framework (per-plugin)                                                                                                                                                                                                              | Private/public workspaces; [granular roles and permissions](https://stoplight.io/api-documentation) per project                                                               |
| **Git hosting support**          | GitHub, GitHub Enterprise, GitLab, GitLab Enterprise, Bitbucket, Azure DevOps, Gerrit, Gitea, AWS CodeCommit, Harness Code ([full list](https://backstage.io/docs/next/features/techdocs/))                                                                      | GitHub, GitLab, Bitbucket ([Stoplight docs](https://docs.stoplight.io/))                                                                                                      |
| **Plugin / extension model**     | Large [MkDocs plugin ecosystem](https://www.mkdocs.org/user-guide/plugins/) + [TechDocs Addon Framework](https://backstage.io/blog/2022/05/13/techdocs-addon-framework) for UI addons                                                                            | Stoplight Elements (embeddable); limited extensibility outside the platform                                                                                                   |
| **Linting / style enforcement**  | External (e.g. markdownlint, ruff); no native enforcement                                                                                                                                                                                                        | [Spectral](https://stoplight.io/open-source/spectral) for OpenAPI linting; style guides for API design consistency                                                            |
| **Best fit**                     | Internal engineering docs: runbooks, architecture notes, service onboarding, ownership metadata                                                                                                                                                                  | API consumers (internal or external): endpoint reference, schema explorer, quickstart guides                                                                                  |

## The broader landscape

**Developer documentation portals** are platforms that publish technical docs (API reference, guides, runbooks) for developers to discover and integrate with software. Two rough subtypes:

- **Internal developer portals** (IDP): service catalog + internal team docs. Backstage is the dominant open-source option.
- **API docs portals**: public or partner-facing API reference + interactive try-it-out. Stoplight, ReadMe, Mintlify occupy this space.

Some tools span both.

| Platform | Type | Model | Notable for |
|---|---|---|---|
| [Backstage](https://backstage.io) (Spotify) | IDP | Open-source, self-hosted | Service catalog, TechDocs, plugin ecosystem |
| [Stoplight](https://stoplight.io) | API docs portal | SaaS (now SmartBear) | OpenAPI-first, Spectral linting, try-it-out |
| [ReadMe](https://readme.com) | API docs portal | SaaS | Personalized docs with live API keys, developer metrics |
| [Mintlify](https://mintlify.com) | API docs portal | SaaS | MDX-based, Git-synced, instant deploys, modern UI |
| [Fern](https://buildwithfern.com) | API docs + SDK gen | SaaS/OSS | Generates typed SDKs (TS, Python, Java, Go) from OpenAPI |
| [GitBook](https://gitbook.com) | Docs portal | SaaS | Git-native, good for API docs and knowledge bases |
| [GitHub Pages](https://pages.github.com) | Static hosting | Free, self-built | Bare hosting — you supply the SSG (Jekyll, MkDocs, etc.) |
| [MkDocs + Material](https://squidfunk.github.io/mkdocs-material/) | SSG | OSS | What Backstage TechDocs uses under the hood; self-hosted |
| [Docusaurus](https://docusaurus.io) | SSG | OSS (Meta) | React-based, strong versioning, commonly hosted on GitHub Pages |
| [Redocly](https://redocly.com) | API docs portal | SaaS/OSS | OpenAPI-first like Stoplight; strong multi-file OpenAPI bundling |
| [Zuplo / Zudoku](https://zuplo.com) | API gateway + docs | SaaS | Built-in API key self-service alongside docs |
